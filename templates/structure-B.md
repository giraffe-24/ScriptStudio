# 台本テンプレート B：機能紹介型

> 新機能や知られていない機能の始め方・使い方を紹介する型
> 例：Find Hub、Googleセキュリティ設定

---

## 適用条件

```
・新しいサービス/機能の紹介
・「知らない人が多い」ものを教える
・始め方から使い方まで一通り見せる
・複数の機能を順番に紹介する
```

---

## 構成

```
【導入】約400字（10-15%）
  │
  ├── 自己紹介（固定）
  ├── 問いかけフック「皆さんは〜したことありますか？」
  ├── 現状説明「〜は〜ですが、〜は...」
  ├── 解決提示「〜を使えば〜ができます」
  ├── 価値リスト（できること3-8個）
  └── 宣言「ってことで今回は〜を紹介します」

【本題】約2600-3600字（70-75%。合計3001〜4200字）

  │
  ├── 〇〇の始め方
  │     ├── 前提条件（必要なもの、確認事項）
  │     ├── インストール/設定手順
  │     └── 初期設定完了の確認
  │
  ├── 〇〇の設定
  │     ├── 重要な設定項目
  │     ├── 設定の意味（なぜこの設定か）
  │     └── おすすめ設定
  │
  ├── 〇〇を使ってみる
  │     ├── 基本機能の実演
  │     └── 「こんな風に〜」で効果を見せる
  │
  ├── 〇〇で△△する（機能別に複数）
  │     ├── 機能の説明
  │     ├── 使い方
  │     └── 注意点
  │
  └── 〇〇の注意点（番号付き）
        ├── 注意点①：〜
        └── 注意点②：〜

【まとめ】約300字（10-15%）
  │
  ├── 価値の再確認「〜があれば〜できます」
  ├── 具体的なメリット列挙
  ├── 行動喚起「是非〜を使ってみてください」
  └── 定型締め（固定）
```

---

## 導入テンプレート

```
「楽」を極めて最大の結果を出す、効率化オタクのあらきりです。

皆さんは【日常のシチュエーション】、どうしますか。

最近の【比較対象】はかなり便利で、【できること】がわかります。
対して【今回紹介するもの】はそういった機能が【現状】。

ですが、【サービス名】を使えば、【できること1】や【できること2】などが可能になります。

【スライド：できることリスト】
・〇〇
・〇〇
・〇〇
（3-8個）

ってことで今回は、【サービス名】の始め方、使い方を紹介します。
```

---

## 本題テンプレート

### 「始め方」セクション

```
【見出し】〇〇の始め方

【画面説明】
〇〇のPC画面はこんな感じで、【表示されているもの】が表示されてます。

【前提確認】
では早速〇〇をインストールしていきます。
まずはじめに、【前提条件】を確認しましょう。
これが【問題の状態】だと〇〇の本来の力を発揮できないので注意してください。

【確認手順】
【前提条件】の確認方法は、【手順1】をタップしてください。
そうすると【表示】ができるので、【推奨設定】にしておきましょう。

【インストール手順】
続いて、【ストア名】から、〇〇を探します。
インストールボタンを押してください。
インストールが完了したらアプリを立ち上げます。
【初期設定の手順】を進めます。
これでインストールが完了しました。
```

### 「設定」セクション

```
【見出し】〇〇の設定（△△を〇〇する）

【設定画面への移動】
次に、〇〇の設定をしていきます。
【操作】をタップします。
【次の操作】を押してください。

【設定の説明】
少しスクロールすると、【設定項目】を設定できます。

【なぜの説明】
これ、よく見ると少し不思議ですよね。
どうして【疑問】なのか。
実は、〇〇は【仕組みの説明】を採用しています。
【詳しい説明】。

【おすすめ設定】
なので、個人的には【おすすめ設定】が良いと思いますが、
【別の考慮事項】って人は【代替案】にしたりするのが良いと思います。
```

### 「機能紹介」セクション

```
【見出し】〇〇で△△する

【導入】
次に、【機能名】を試してみます。

【操作】
まずは【操作1】から試してみます。
すぐさま【結果1】と表示され【結果2】に変わりました。

【実際の動作】
【操作】したときの〇〇はこんな感じでした。
【動作の詳細】。
そのため、【メリット】と思います。
```

### 「注意点」セクション

```
【見出し】〇〇の注意点①【注意内容】

〇〇にはいくつかの注意点があります。

まず、【注意点の概要】です。
【詳しい説明】。

【対処法/代替案があれば】
【対処法の説明】。

【見出し】〇〇の注意点②【注意内容】

【注意点の概要】です。
【詳しい説明】。

【見分け方/確認方法】
【状態A】のときは【表示A】。
【状態B】のときは【表示B】と表示されるので、ここで見分けることができます。
```

## まとめテンプレート

```
【価値の再確認】
〇〇があれば、万が一【シチュエーション】してもすぐに対応できます。

【具体的メリット】
【メリット1】しても、【機能1】すればすぐに見つかります。
【シチュエーション】できるので、〇〇を入れておくだけでもしもの時にも落ち着いて行動できます。

【社会的背景（任意）】
最近は、【最近の傾向/ニュース】なんて話も耳にするので、日頃から備えておくに越したことはありません。

【行動喚起】
是非、〇〇を使ってみてください。

【定型締め】
このチャンネルでは「楽」を極めるために私が検証し、本当に効果があった仕組みだけを紹介しています。

大人の教養として『賢くサボる』を知りたい方は、ぜひチャンネル登録と高評価をお願いします。

日常に潜む「無駄な手間」や｢これも知りたい｣などのリクエストも、遠慮なくコメントで教えてください。

それでは、また次回の動画でお会いしましょう。さようなら。
```

## この型の成功パターン

### Find Hub台本から抽出

```
【効果的だったフレーズ】
・「これがオフになっているとFindHubの本来の力を発揮できないので注意してください」
・「これ、よく見ると少し不思議ですよね。どうして〜なのか」
・「さすがに現在位置を動画で見せることはできないので」（リアリティ）

【構成のポイント】
・「始め方」→「設定」→「使ってみる」の順で段階的に
・各機能を独立したセクションで紹介
・注意点は最後にまとめて、番号付きで
・「なぜ」の説明を入れて理解を深める
```
