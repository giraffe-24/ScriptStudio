最短で始める

パソコンにこのフォルダ（YT_TalkScript）を用意し、その中でAIアシスタント（例：Cursorのチャットなど）と会話する想定です。特別なインストール作業は不要です。

やりたいこと：決まっている

/全工程 あなたのテーマをここに書く
  1. テーマについて案が返ってくるので、進めてよさそうな案をひとことで選ぶ。
  2. 同様に構成から台本、タイトル案が順に進み、最後にチェックが走ります。

やりたいこと：自分のペースで進めたい

/企画 テーマ
（案を選んだあと）

/構成
/執筆
/タイトル
/チェック

各ステップのあと、「この案で」「ここだけ直したい」のように短文で伝えれば大丈夫です。

よく出る言葉（用語）

切り口 … 動画全体の売り込み方向（例:「ズボラ向け」「時短」「よくある失敗から説明」など）。最初に三つほど提案され、どれかを選ぶ流れになります。
台本タイプ A / B / C … 「一つのツールを手順で」「機能の紹介」「まとめて〇〇選」の三種類。細かく知らなくても構成の段でAIが決めます。
SSOT（単一の正）config/ にあるブランド・文体などのルール。普段は触らなくて構いません。
成果物フォルダ … 過程や最終のアウトプットが outputs/日付とテーマ名/ に並びます（企画〜レビューの番号付きファイル）。

コマンド一覧

チャット入力欄にそのまま打ち込みます。角かっこ [] は書かず、説明どおり自分の文言に差し替えてください。

メインコマンド
入力例 何がおこるか
/全工程 Gmail整理術 企画からチェックまで一気通貫
/企画 Googleカレンダー活用 切り口などの案が三つ並ぶ
/構成 承認済みの切り口から、見出しと流れが設計される
/執筆 台本本編が書き上がる
/タイトル タイトル案・サムネ文言案など
/チェック お約束どおり話せているか、スコア付きレビュー

補助の言い回し（一般文でも可）

表だけに縛られる必要はありません。話すような日本語で依頼すれば大丈夫です。

伝えたいこと 入力の例
一部だけ直したい 導入の最初の段落をもっと短くして
今のステップをやり直したい /構成 を最初からやり直して

作業の流れ（ざっくり）

順序 あなた AI側
1 テーマまたは切り口の選択 企画書を出す（競合視点など含む）
2 OKまたは修正希望 構成案・時間の目安
3 問題なければ次へ 台本
4 好みのタイトル・文言を選択 タイトルとサムネ案
5 — 品質チェック（スコアと改善案）

合格ラインなど細かな基準は CLAUDE.md と config/quality.md にあります。このHTMLページでは省略しています。

ファイルはどこにできる？

outputs/
└── YYYYMMDD-テーマ名/
    ├── 01-plan.md      … 企画
    ├── 02-structure.md … 構成
    ├── 03-script.md    … 台本
    ├── 04-title.md     … タイトル・サムネ案
    └── 05-review.md    … レビュー

examples/ にうまくいった過去稿を置くと参照されやすくなります（任意）。

困ったとき

うまく言えなくて伝わらない

「抽象的に良くして」より、場所・用途・時間が分かると改善しやすいです。例：STEP2だけ画面タップ順を増やしたい／8分収録向けにもう少し短く、など。

途中で話の方向を変えたい

「実は〇〇側に軸を移したい」と伝え、そのあとそのステップをもう一回実行して構いません。

詳しい仕様を知りたい

MANUAL.md が長めのテキストガイド、CLAUDE.md がシステム仕様です。ここは読みやすさ優先の要約です。

詳細ガイド: 同フォルダ内の MANUAL.md（Markdown）。
