最短で始める

パソコンにこのフォルダ（ScriptStudio）を用意し、その中でAIアシスタント（例：Cursorのチャットなど）と会話する想定です。特別なインストール作業は不要です。CLAUDE.md が運用への入口になり、コマンド表やワークフロー一覧は docs/orchestration.md にあります。

やりたいこと：決まっている

/全工程 あなたのテーマをここに書く
  1. テーマについて案が返ってくるので、進めてよさそうな案をひとことで選ぶ。
  2. 同様に構成から台本、タイトル案が順に進み、最後にチェックが走ります。

やりたいこと：自分のペースで進めたい

/テーマ調査 （任意／テーマ未定なら）

/企画 テーマ
（案を選んだあと）

/構成
/執筆
/タイトル
/チェック
/推敲比較 （任意・台本に確定稿をマーカーで区切って載せたあと）

各ステップのあと、「この案で」「ここだけ直したい」のように短文で伝えれば大丈夫です。

テーマが未定のとき（/テーマ調査）

ネタだけ先に集めてから企画するときです。詳細仕様は agents/scout.md にあります。

  1. /テーマ調査（任意で /テーマ調査 Gmail のように焦点を一言）
  2. A / B / C のメニューが先だけ出るので、選んで返信（返信があるまで一覧はださない）
  3. 調査結果はチャットと、outputs/00-discovery.md に残る（必要なら別名で保存）
  4. 採用テーマが決まったら /企画 そのテーマ
A / B / C の意味：A＝説明できる定番寄り／B＝ニュース・アップデート・旬寄り／C＝半々。

よく出る言葉（用語）

切り口 … 動画全体の売り込み方向（例:「ズボラ向け」「時短」「よくある失敗から説明」など）。最初に三つほど提案され、どれかを選ぶ流れになります。
台本タイプ A / B / C … 「一つのツールを手順で」「機能の紹介」「まとめて〇〇選」の三種類。細かく知らなくても構成の段でAIが決めます。
SSOT（単一の正）config/ にあるブランド・文体などのルール。索引は config/README.md。普段は触らなくて構いません。
テーマ調査/テーマ調査 メニューを選んだあとターゲットに合うテーマ候補を数個出します。詳細な手順・禁止事項は scout.md 側です。
成果物フォルダ … 過程や最終のアウトプットはリポジトリ直下の outputs/ に並びます。調査をした回は 00-discovery.md が付くことがあります。複数案件を並行するときはファイル名で区別してください。

コマンド一覧

チャット入力欄にそのまま打ち込みます。角かっこ [] は書かず、説明どおり自分の文言に差し替えてください。

入力例 何がおこるか

作業の流れ（ざっくり）

順序 あなた AI側
0（任意） /テーマ調査 で A/B/C を選ぶ／焦点を伝える テーマ候補と調査ノート（ファイル保存）
1 テーマまたは切り口の選択 企画書を出す（競合視点など含む）
2 OKまたは修正希望 構成案・時間の目安
3 問題なければ次へ 台本
4 好みのタイトル・文言を選択 タイトルとサムネ案
5 — 品質チェック（スコアと改善案）
6（任意） /推敲比較（台本内に <<<SCRIPTSTUDIO_CALIB_SPLIT>>> で確定稿を区切っていること） 差分から学び・SSOT への改善提案（チャット。ファイルに残すときは「06-calibration.md に保存」と明示）

合格ラインなど細かな基準は config/quality.md と agents/reviewer-rubric.md にあります。コマンド早見は docs/orchestration.md。このHTMLページでは省略しています。

推敲比較・仕様の参照先・開発ツール

推敲比較（/推敲比較）は、台本ファイルの中で <<<SCRIPTSTUDIO_CALIB_SPLIT>>> より上をAI原稿、下を手元の確定稿として並べたうえで依頼します。マーカーは config/calibration.md、手順は agents/calibrator.md、ルールは config/quality.md の「推敲比較」の節を参照してください。Cursor では .cursor/commands/推敲比較.md からも同じ意図で呼び出せます。

仕様を深く読むとき：表形式のコマンド一覧と全体フローは docs/orchestration.md。運用の短文入口は CLAUDE.md。ブランド・文体・品質のSSOTは config/README.md を手がかりに各 config/*.md へ。

開発用ツール（任意）：タイトル案のローカルUIは npm run title-studio（手順は docs/orchestration.md）。台本を印刷向けHTMLにするには npm run script-doc -- outputs/03-script.md のようにパスを渡します。

困ったとき

うまく言えなくて伝わらない

「抽象的に良くして」より、場所・用途・時間が分かると改善しやすいです。例：STEP2だけ画面タップ順を増やしたい／8分収録向けにもう少し短く、など。

途中で話の方向を変えたい

「実は〇〇側に軸を移したい」と伝え、そのあとそのステップをもう一回実行して構いません。

詳しい仕様を知りたい

長めの手順書は MANUAL.md（Markdown）。コマンド一覧とワークフロー表は docs/orchestration.md。運用の入口は CLAUDE.md（短文）。SSOT は config/README.md 経由で各 config/*.md。/テーマ調査 の詳細だけは agents/scout.md を参照してください。ここは読みやすさ優先の要約です。

/テーマ調査 がすぐリストを出してこない

仕様です。最初は A/B/C のメニューだけが出ます。選択を返したあとに調査結果が進みます。
