## 導入（約0:45～1:00）

- 固定自己紹介（`voice.md` / `brand.md`）。

- **今日の約束**：付箋の便利機能総ざらいではなく、**同期して同じメモ一覧が見える状態**になるところまで。

## セクション設計（本題）

### 1. 非ゴールの宣言（約0:30）

- **やらないこと**：メモ術、色設計講義、OneNote／他アプリ論争、キーボードショートカット大全。
- **ゴール**：PC の付箋に書いた内容が **Android 上の公式の経路**で一覧できるかを確認できる。

※アプリ名・メニュー名は収録時点のストア・アプリ画面に合わせて差し替え（UI変更に強い話し方：「画面上の〇〇という名前」を指示）。

### 2. PC側（Windows）｜チェックリスト（約2:00～2:20）

| # | 内容（話す／見せる） |
|---|----------------------|
| P1 | 付箋（Sticky Notes）を開く場所まで（スタートメニューや検索など、自分の環境で一パターンに固定）。 |
| P2 | **Microsoft アカウント**でサインインしている状態か（設定画面または付箋内のインジケータ。**断定は自分の画面上で** と言える範囲）。 |
| P3 | テスト用に **短文を1つ** 新規付箋で書いておく（あとから Android でそれが見えるか確認する用）。 |

### 3. Android側｜セットアップ（約2:30～2:45）

| # | 内容（話す／見せる） |
|---|----------------------|
| A1 | **Google Play** から Microsoft 提供の **付箋（Sticky Notes）** アプリを導入（アプリ表示名はストア準拠で読み上げ）。 |
| A2 | 同じ **Microsoft アカウント**でサインイン（PC と同一であることを口述で強調）。 |
| A3 | アプリを開いた直後に **リスト／メモ一覧** が表示される状態まで。権限ダイアログが出たら **必要最小限のみ** と一言（過剰に怖がらせない）。 |
| A4 | 端末により **自動同期までに少しタイムラグがある** と正直に伝える（過度な秒数断言はしない）。 |

### 4. 同期の確認・試し（約0:45～1:00）

- Android の一覧に、PC で書いた **テスト短文** が出るか。
- または Android で追記／PC側を引きなおして確認（**どちらか一方の方向だけ**でもよい／尺とのトレードオフ）。

### 5. うまくいかないとき｜チェックだけ（約0:35～0:45）

**最大3項目** に絞る（ここだけ尺を死守）。

1. **PC と Android が同じ Microsoft アカウントか**（双方の設定画面・メールの下4桁など、映せる範囲）。  
2. **アプリと Windows のアップデート**（「一度ストア／Windows Update を自分の画面上で確認」レベル。断定はしない）。  
3. **サインインの切れ** — どちらかで再ログイン（手順は1画面のみ）。

競合サービスとの比較や「〜だから危ない」系は入れない（品質）。

### 6. まとめ〜定型締め前（約0:35）

- **付箋は短期メモ向き** であり、資産保管の全部を託さない一文（過信させない）。
- **今日できた状態**の復唱（リストが見える＝OK）。
- 定型締め（`brand.md` の固定文をそのまま）。
