# 企画書：【デモ】効率化アプリ100個を1週間で全部試して、最強の1個を決める

> ⚠️ このエピソードは**公開レビュー用のデモ**です。MrBeast式の企画フォーマット（大量検証→足切り→1つに絞る）を効率化ジャンルに応用したサンプルで、登場するアプリ名・数値・エピソードはすべて架空です。

---

## 企画意図

アプリ紹介動画の定番「おすすめ10選」は、結局視聴者に選択を委ねてしまい、悩みを解決しない。
MrBeastの動画が強いのは「規模の大きい検証」と「脱落していく過程のエンタメ化」、そして**結論を1つに絞り切る**こと。この構造を効率化ジャンルに移植する。

| 項目 | 内容 |
|------|------|
| フォーマット | 大量検証 → 段階的な足切り → 優勝発表 |
| 規模感 | アプリ100個 × 1週間 |
| 山場 | 最終発表（あえて一番地味なアプリが優勝する意外性） |
| 持ち帰り | 視聴者自身がアプリを選べる「3つの基準」 |

## 想定視聴者

効率化アプリに興味はあるが、種類が多すぎて選べずにいる20〜50代。
「インストールしては放置」を繰り返しており、比較表よりも**結論だけ教えてほしい**と思っている。

## 検証ルール（3つの足切り基準）

1. **起動3秒ルール** — 起動して3秒で使い方が分かるか
2. **設定5分ルール** — 初期設定が5分以内に終わるか
3. **3日後ルール** — 3日後、自分が自然にまだ使っているか

## 構成（脱落の流れ）

```
100個 ──1日目──▶ 52個 ──3日目──▶ 17個 ──7日目──▶ 3個 ──▶ 優勝1個
        起動3秒          設定5分＋自然消滅        ファイナリスト
```

## 結論（優勝）

架空のメモアプリ**「サッとメモ（仮）」**。
理由：「考える前に起動が終わっている」。派手な機能より**摩擦の少なさ**が継続を決める。

## 差別化ポイント

- 競合の「おすすめ10選」に対し、**1個に絞り切る**
- 脱落モンタージュでエンタメ性を確保しつつ、基準は3つだけに絞って実用性を残す
- 「あなたが100個試す必要はありません」という視聴者への価値の言語化
