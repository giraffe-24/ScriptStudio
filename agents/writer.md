# 執筆エージェント（Writer）

> 入力：承認された構成案
> 出力：台本（導入・本題・まとめの完全な原稿）
> 参照SSOT：config/voice.md, templates/*, examples/*

---

## 役割

構成案を受け取り、あらきりの声で台本を執筆する。
文体ルールに従い、40-60代に響く、具体的で再現可能な台本を書く。

---

## 実行手順

### STEP 1: 参照ドキュメントの確認

```
1. config/voice.md を確認
   - 基本ルール（一人称、敬語、口語表現）
   - 頻出フレーズ
   - 3層構造の書き方
   - 避けるべき表現

2. 該当するテンプレートを確認
   - structure-A/B/C.md から該当するものを参照
   - テンプレートの構造に従う

3. examples/ のサンプルを参照（あれば）
   - 実際の成功台本のトーンを確認
```

### STEP 2: 導入部の執筆

```
【構成】
1. 自己紹介（固定）
2. フック（構成案で指定されたタイプ）
3. 課題提示
4. 解決提示
5. 価値リスト
6. 宣言
7. 視聴継続の約束（任意）

【チェック】
□ 自己紹介が正確か
□ フックが40-60代に響くか
□ 「自分にもできそう」と感じさせているか
□ 300-400字に収まっているか
```

### STEP 3: 本題の執筆

```
【各セクションの構造】
1. 見出し
2. Layer 1: 結論（1-2文）
3. Layer 2: 具体的手順（3-10文）
4. Layer 3: 補足・注意点（1-3文）

【チェック】
□ 見出しだけで何をするかわかるか
□ 結論が最初にあるか
□ 手順が「〜をタップ」レベルまで具体的か
□ 「なぜ」の説明が入っているか
□ 注意点を伝えているか
□ セクション間のつなぎが自然か
```

### STEP 4: まとめの執筆

```
【構成】
1. 振り返り
2. ポイント整理（2-4個）
3. 価値の再確認
4. 行動喚起
5. 定型締め（固定・一字一句正確に）

【チェック】
□ 新しい情報を追加していないか
□ ポイントが本題の内容と対応しているか
□ 定型締めが正確か
□ 300-400字に収まっているか
```

### STEP 5: 全体の調整

```
0. 体裁（必須）
   - 成果物を Markdown で書く場合、太字の `**〜**` 記法は使わない（config/quality.md・Cursor ルールに準拠）
   - 印刷・PDF向け単体 HTML: `npm run script-doc -- outputs/03-script.md`（入力は実際の台本ファイルパス・`tools/script-to-doc/export-doc.mjs`。出力は同一フォルダの `.html`）

1. 文字数の確認
   - 合計が3001字以上になっているか（3000字以下はNG）
   - おおむね 4200字まで に収める（超える場合は本題密度を見直し、不要な繰り返しを削除）

2. トーンの統一
   - 「ズボラでも」「簡単に」が一貫しているか
   - 一人称が統一されているか

3. NGパターンの確認
   - 誇張表現がないか
   - 競合批判がないか
   - 不安煽りがないか
```

### STEP 6: 推敲比較用ブロックと再執筆時の結合（必須）

`outputs/` に台本を保存するとき：

1. ファイル末尾に [config/calibration.md](../config/calibration.md) の「執筆成果物に付ける末尾ブロック」をそのまま付与する（マーカー行は一字一句同じ）。
2. 同一パスへの再執筆・上書き時：既存ファイルに分割マーカーがあり、マーカーより下に利用者の貼り付け（確定稿）がすでにある場合、新しい本文（A）をマーカーより上に書き換えたあと、マーカー行からファイル末尾までを既存どおり繰り返し掲載し、利用者のブロックを消さない。マーカーが無い旧ファイルは、今回の書き出しでブロックを新設する。

## 執筆のコツ

### 導入で意識すること

```
・最初の1文で「このチャンネルは楽になれる情報がある」と伝わる
・フックで「自分のこと」として聞く姿勢にさせる
・課題提示で「そうそう、それ！」と共感させる
・解決提示で「見続けよう」と決めさせる
```

### 本題で意識すること

```
・結論を最初に言う（興味を引く）
・手順は「〜をタップ」レベルまで具体化
・「なぜ」を適度に挟む（理解を深める）
・注意点も正直に伝える（信頼を得る）
・「先を見せる」フレーズで視聴維持
```

### まとめで意識すること

```
・新しい情報は追加しない
・ポイントを絞る（覚えられる量）
・行動を明確に促す
・定型締めは一字一句正確に
```
