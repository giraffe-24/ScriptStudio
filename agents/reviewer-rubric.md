# レビュー用ルーブリック（手続き・採点）

> 本ファイルは SSOT `config/quality.md` の NG・推敲比較・製品事実の方針を、**どう機械的に検査するか**だけを定める。  
> `/チェック` 実行時は [agents/reviewer.md](reviewer.md) と本ファイルをセットで読む。

---

## 品質チェックリスト

### 導入部チェック（10項目）

```
□ 1. 自己紹介が正確か
     「『楽』を極めて最大の結果を出す、効率化オタクのあらきりです」

□ 2. フックが40-60代に響く表現か
     - 問いかけ型 or 共感型 or 挑発型（適度）

□ 3. 視聴者の課題/不満を言語化しているか
     - 「〜が面倒」「〜で困っている」

□ 4. 解決策を明示しているか
     - 「〜を使えば」「でも安心してください」

□ 5. 価値リストがあるか（3-8個）
     - 動画を見る理由を複数提示

□ 6. 宣言があるか
     - 「今回は〜を紹介します」

□ 7. 「自分にもできそう」と感じさせているか
     - 「ズボラでも」「簡単に」

□ 8. 不安を煽っていないか
     - NGパターンに該当しないか

□ 9. 誇張表現がないか
     - NGパターンに該当しないか

□ 10. 30秒以内に読める長さか
      - 約300-400字
```

### 本題チェック（12項目）

```
□ 1. セクション見出しが明確か
     - STEP形式 or 番号形式

□ 2. 「現在地」がわかる構成か
     - 「STEP1」「1.」などの番号

□ 3. 各セクションが3層構造か
     - 結論→手順→補足

□ 4. 結論が最初にあるか
     - 「〜は、〜ができる機能です」

□ 5. 操作が「〜をタップ」レベルまで具体化されているか
     - 抽象的な「設定を変更する」はNG

□ 6. 「なぜ」の説明が適度に入っているか
     - 手順だけでなく理由も

□ 7. 注意点・デメリットを伝えているか
     - 「ただし〜には注意」

□ 8. 専門用語に説明を添えているか
     - 初出時に必ず説明

□ 9. 「先を見せる」テクニックがあるか
     - 「後半で説明します」など

□ 10. 1セクションが長すぎないか
      - 目安：200-400字/セクション

□ 11. セクション間のつなぎが自然か
      - 「続いて〜」「次に〜」

□ 12. NGパターンに該当しないか
      - 誇張、煽り、マウント等
```

### まとめチェック（8項目）

```
□ 1. 振り返りがあるか
     - 「というわけで今回は〜を紹介しました」

□ 2. ポイントが2-4個に整理されているか
     - 多すぎると記憶に残らない

□ 3. 価値を再確認しているか
     - 「〜があれば〜ができます」

□ 4. 行動喚起があるか
     - 「是非試してみてください」

□ 5. 定型締めが正確か
     - 一字一句正確に

□ 6. 短くまとまっているか
     - 約300-400字

□ 7. 新しい情報を追加していないか
     - まとめで新情報はNG

□ 8. NGパターンに該当しないか
```

### 全体チェック（10項目）

```
□ 1. トーンが一貫しているか
     - 「ズボラでも」「簡単に」

□ 2. 40-60代に伝わる言葉遣いか
     - 若者言葉を使っていないか

□ 3. 文字数が基準か
     - 3001字以上である（3000字以下はNG）
     - 目安〜4200字。大幅超過する場合は冗長・離脱リスクとして要確認
     - 5-10分相当の読み上げ体感と矛盾していないか

□ 4. 導入:本題:まとめの比率が適切か
     - 約10-15% : 70-75% : 10-15%

□ 5. 一人称が統一されているか
     - 「私」or「わたし」

□ 6. 敬語レベルが統一されているか
     - です/ます調

□ 7. 操作説明のデバイスが統一されているか
     - PC or スマホ

□ 8. 「ズボラでもできる」感が伝わるか
     - 難しそうに見えないか

□ 9. 実際に5-10分で読み上げられるか
     - 声に出して確認

□ 10. ブランドコアに沿っているか
      - 「楽して結果を出す」
```

## 品質スコアリング

### 採点基準

| スコア | 基準 |
|--------|------|
| A（90-100点） | 即使用可能。修正不要 |
| B（70-89点） | 軽微な修正で使用可能 |
| C（50-69点） | 部分的な書き直しが必要 |
| D（50点未満） | 大幅な書き直しが必要 |

### 減点基準

| 項目 | 減点 |
|------|------|
| NGパターン該当（1件につき） | -15点 |
| 導入チェック未達（1項目につき） | -3点 |
| 本題チェック未達（1項目につき） | -2点 |
| まとめチェック未達（1項目につき） | -3点 |
| 全体チェック未達（1項目につき） | -2点 |

### 合格基準

```
・スコア70点以上（B以上）
・NGパターン該当が0件
・導入の必須5項目（1,3,4,5,6）がすべて達成
・定型締めが正確
```

```
- 本題: …
```
