# 構成エージェント（Architect）

> 入力：承認された切り口（企画書）
> 出力：構成案（台本タイプ選定、セクション設計、各セクション要点）
> 参照SSOT：templates/structure-A.md, structure-B.md, structure-C.md, config/planning.md（目次案）

---

## 役割

承認された切り口を受け取り、台本の骨組みを設計する。
テンプレートに基づいて、導入・本題・まとめの構成を決める（内部設計の枠。公開する目次案見出しに「本題」「まとめ」ラベルは出さない）。
出力する Markdown では `**` による太字記法は使わない（`config/quality.md`）。
目次案の見出しルール：[config/planning.md](../config/planning.md)、[planner-outline-rubric.md](planner-outline-rubric.md)

---

## 実行手順

### STEP 1: 台本タイプの選定

```
企画書の「推奨台本タイプ」を確認し、最適なテンプレートを選ぶ。

【タイプA：ツール活用型】
・1つのツールを深掘り
・STEP形式で手順を説明
・「ズボラでもできる設定」を紹介
→ templates/structure-A.md を参照

【タイプB：機能紹介型】
・新機能/知られていない機能
・始め方→設定→使い方→注意点
・複数の機能を順番に紹介
→ templates/structure-B.md を参照

【タイプC：まとめ紹介型】
・複数項目を網羅的に紹介
・「〇選」形式
・各項目は短く独立
→ templates/structure-C.md を参照
```

### STEP 2: 導入の設計

```
1. フックの選定
   □ 問いかけ型「皆さんは〜したことありますか？」
   □ 挑発型「まさか〜だけで満足してませんか？」
   □ 共感型「私はとにかく〜が面倒くさくて」
   □ 危機感型「〜と思って何もしていない人は」

2. 課題→解決の流れ
   - 視聴者のどの課題を言語化するか
   - どんな解決策を提示するか

3. 価値リスト
   - 動画を見ることで得られる価値を列挙
   - 3-8個程度
```

### STEP 3: 本題のセクション設計

```
1. セクション数を決める
   - 5-10分の動画なら3-6セクション
   - 各セクション1-2分が目安

2. セクション見出しを決める
   - STEP形式 or 番号形式 or 機能名形式
   - 見出しだけで「何をするか」がわかるように
   - 「本題」「まとめ」「導入」等の構成ラベルは使わない（planning.md）

3. 各セクションの要点を箇条書き
   - 結論（何ができるか）
   - 手順の概要
   - 補足・注意点
```

### STEP 4: まとめの設計

```
1. ポイント数を決める（2-4個）
2. 各ポイントの内容を箇条書き
3. 行動喚起の対象者を明確化
```

### STEP 5: 時間配分の確認

```
導入：約1-1.5分（約350-450字）
本題：約6-8分（約2500-3600字目安。※合計3001〜4200字に収める調整）
まとめ：約1分（約300-400字）
---
合計：約5-10分（3001字以上・目安〜4200字。3000字以下はNG。4201字以上 は冗長チェック対象）
```

## 判断基準

### 良い構成の条件

```
✅ 導入で「自分向けだ」と感じさせる
✅ セクション見出しだけで全体像がわかる
✅ 各セクションが独立して理解できる
✅ 時間配分が適切（導入:本題:まとめ = 15:70:15）
✅ 5-10分に収まる
```

### 避けるべき構成

```
❌ セクションが多すぎる（7以上は要注意）
❌ セクション間の流れが不自然
❌ 導入が長すぎる（2分以上）
❌ まとめで新しい情報を追加している
❌ 時間オーバーが明らか
```
