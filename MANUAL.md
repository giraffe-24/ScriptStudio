# YouTubeトークスクリプト作成システム マニュアル

> 最終更新: 2026-05-03
>
> このマニュアルは「あらきりチャンネル」のYouTube台本を作成するためのガイドです。
> 初めての方でも迷わず使えるよう、ステップバイステップで説明します。

---

## マニュアルをどう保全するか

- 自動では追記・修正しません。`CLAUDE.md`・`config/`・`agents/` などほかが変わっても、このファイルとブラウザ用の `manual/index.html` は、その都度ひっつきません。
- 更新したくなったら、その旨だけ明示的に依頼してください（例:「マニュアルを最新仕様に合わせて」）。
- 仕様の正（SSOT）は引き続き `CLAUDE.md` と各 `config/*.md` です。細かなルール確認はそちらを参照してください。
- ブラウザで読みたいときは、リポジトリ内の [`manual/index.html`](manual/index.html) をダブルクリックまたは「ファイルとして開く」で表示できます（オフライン可）。

## 1. このシステムでできること

### 一言で言うと

テーマを伝えるだけで、あらきりスタイルの台本が完成します。

### 具体的には

| できること | 説明 |
|------------|------|
| テーマ候補の調査 | ターゲットに沿うネタ案を複数、その根拠とともに洗い出し（`/テーマ調査`） |
| 企画立案 | テーマから3つの切り口を提案 |
| 構成設計 | セクション分け、時間配分を設計 |
| 台本執筆 | あらきりの声で台本を執筆 |
| タイトル・サムネ | クリック率を高めるタイトル・サムネ案を提案 |
| 品質チェック | NGパターン検出、スコアリング |

### 対応する台本タイプ

| タイプ | 説明 | 例 |
|--------|------|-----|
| A: ツール活用型 | 1つのツールを深掘り | Gmail整理術 |
| B: 機能紹介型 | 新機能の使い方 | Find Hub |
| C: まとめ紹介型 | 複数項目を紹介 | 〇〇8選 |

## 2. 始める前に

### 必要なもの

- Claude Code（または Cursor など、このフォルダで動かすチャット環境）
- テーマが決まっていれば、そのキーワード（まだでも `/テーマ調査` で候補を出せる）

### 初回セットアップ

特別な設定は不要です。このフォルダでClaude Codeを起動すれば、自動的にCLAUDE.mdが読み込まれます。

## 3. 基本的な使い方

### パターン1: 一気通貫で作成する（おすすめ）

最もシンプルな方法です。テーマを渡すだけで、企画から完成まで進みます。

```
/全工程 Googleドライブの整理術
```

流れ:
1. AIが企画案を提示 → あなたが選択
2. AIが構成案を提示 → あなたが確認
3. AIが台本を執筆 → あなたが確認
4. AIがタイトル・サムネ案を提示 → あなたが選択
5. AIが品質チェック → 合格なら完成

### パターン2: ネタだけ先に広げたい（任意）

そもそもテーマが固まっていないとき、アイデア幅を広げてから `/企画` に進めます。

```
/テーマ調査          ← または /テーマ調査 Gmail など焦点のヒントのみ
↓ 最初に AI が『A/B/C メニュー』だけ出すので、選択して返信
↓ 調査結果は outputs/…/00-discovery.md にも保存される
採用したテーマで /企画
… 以降はパターン1と同様
```

### パターン3: ステップごとに進める

より細かくコントロールしたい場合は、各ステップを個別に実行します。（必要なら手前で `/テーマ調査`。）

```
ステップ0（任意）: /テーマ調査 → メニューを選んでから本調査
ステップ1: /企画 Googleドライブの整理術
ステップ2: /構成
ステップ3: /執筆
ステップ4: /タイトル
ステップ5: /チェック
```

## 4. コマンド一覧

### メインコマンド

| コマンド | 説明 | 入力例 |
|----------|------|--------|
| `/テーマ調査 [任意ヒント]` | テーマ候補を市場観点で洗い出し（先にメニューで調査モードを選ぶ／保存は `00-discovery.md`） | `/テーマ調査` `/テーマ調査 カレンダー` |
| `/全工程 [テーマ]` | 企画から完成まで一気通貫 | `/全工程 Gmail整理術` |
| `/企画 [テーマ]` | 切り口を3案提案 | `/企画 Googleカレンダー活用` |
| `/構成` | 台本の骨組みを設計 | `/構成` |
| `/執筆` | 台本を執筆 | `/執筆` |
| `/タイトル` | タイトル・サムネ案作成 | `/タイトル` |
| `/チェック` | 品質レビュー | `/チェック` |

### 補助コマンド

| コマンド | 説明 |
|----------|------|
| `/修正 [指示]` | 台本の特定箇所を修正 |
| `/再生成` | 現在のステップをやり直す |

## 5. ワークフロー詳細

### STEP 0: テーマ調査（/テーマ調査）（任意）

あなたがやること:
- `/テーマ調査` と打つ（必要なら `Gmail` など焦点を一言だけ添えてよい）。
- AI が最初にだすのは 調査モードの選択だけ（A＝定番寄り／B＝ニュース・旬寄り／C＝半々）。
- メニューへの返信があるまで、一覧や本調査の結果は出てこない想定。A / B / C と、オプションのフォーカスを返す。

AIがやること:
- （メニューを出した後）ターゲット・ブランド（SSOT）に照らした候補 3〜6件 と根拠の整理。
- `outputs/YYYYMMDD-[ラベル]/00-discovery.md` に調査ノートとして保存。
- 切り口3案の確定はしない（それは STEP 1 `/企画`）。

出力の位置づけ:
- 「調査メモ」であり、そのまま台本に使える断定ではありません。次は `/企画 [採用テーマ]` へ。

---

### STEP 1: 企画（/企画）

あなたがやること:
- テーマを伝える

AIがやること:
- 3つの切り口を提案
- 競合分析
- 差別化ポイントの提示

出力例:
```
切り口A（推奨）: ズボラでもできるGoogleドライブ整理術
切り口B: Googleドライブ活用テク5選
切り口C: 容量パンパンを解消！Googleドライブ整理法
```

あなたの判断:
- 「Aでお願いします」など、採用する切り口を伝える

---

### STEP 2: 構成（/構成）

あなたがやること:
- 切り口を承認する

AIがやること:
- 台本タイプ（A/B/C）を選定
- 導入・本題・まとめの設計
- セクション見出しの決定
- 時間配分の確認

出力例:
```
導入（約300字）
├── フック: 共感型
├── 課題: 「ファイルが見つからない」
└── 宣言: 「ズボラでもできる整理術を紹介」

本題（約1800字）
├── STEP1: フォルダ構成をシンプルに
├── STEP2: スター機能を活用
├── STEP3: 検索を使いこなす
└── STEP4: 定期的な整理は不要にする

まとめ（約300字）
```

あなたの判断:
- 「OKです」または「STEP2と3を入れ替えて」など

---

### STEP 3: 執筆（/執筆）

あなたがやること:
- 構成を承認する

AIがやること:
- 導入部の執筆
- 本題の執筆（3層構造）
- まとめの執筆
- 定型締めの追加

出力:
- 完全な台本（3001字以上、目安〜4200字）

あなたの判断:
- 「OKです」または「導入のフックを変えて」など

---

### STEP 4: タイトル・サムネ（/タイトル）

あなたがやること:
- 台本を確認する

AIがやること:
- タイトル案を5つ提案
- サムネイルテキスト案を3つ提案
- 推奨の組み合わせを提示

出力例:
```
タイトル案1（推奨）: ズボラでも3分でできるGoogleドライブ整理術
タイトル案2: Googleドライブぐちゃぐちゃな人必見！簡単整理法
...

サムネ案1（推奨）:
[メイン] ズボラでもできる
[サブ]   Googleドライブ整理術
```

あなたの判断:
- 「タイトル1、サムネ1でお願いします」など

---

### STEP 5: 品質チェック（/チェック）

あなたがやること:
- なし（自動実行）

AIがやること:
- NGパターンの検出
- チェックリストの確認
- スコアリング（100点満点）
- 改善提案

出力例:
```
スコア: 92点
判定: A（合格）

良い点:
- 導入のフックが効果的
- 手順が具体的
- ズボラ感が一貫している

改善提案:
- STEP2の補足がやや長い → 短縮を推奨
```

結果:
- 70点以上 → 合格（完成）
- 70点未満 → 修正後、再チェック

## 6. よくある質問（FAQ）

### Q1: テーマが思いつかない

A: 以下のような切り口で考えてみてください：

```
・最近自分が使って便利だったツール
・視聴者からのリクエスト
・競合チャンネルで伸びている動画のテーマ
・Googleトレンドで上昇中のキーワード
```

### Q2: AIの提案が気に入らない

A: 具体的なフィードバックを伝えてください。

```
❌ 「違う感じで」
✅ 「もっとズボラ感を強調して」
✅ 「STEP1の手順を詳しく」
✅ 「フックを問いかけ型に変えて」
```

### Q3: 途中で方針を変えたい

A: いつでも変更できます。

```
「やっぱり切り口Bで進めてください」
「構成を見直したいので、/構成 をもう一度」
```

### Q4: 過去の台本を参考にしてほしい

A: examples/ フォルダに成功台本を追加すると、AIが参照します。

```
examples/
├── example-calendar.md  ← 成功した台本を保存
└── example-gmail.md
```

### Q5: チェックで不合格になった

A: 指摘された箇所を修正して、再度 `/チェック` を実行してください。

```
/修正 導入のフックを共感型に変更して
/チェック
```

### Q6: `/テーマ調査` って何からすればよい？

A: まず `/テーマ調査` と打つだけでよく、返ってきた A/B/C メニュー に答えます。その後できあがった候補を見て、ひとつのテーマ名を決めたら `/企画 そのテーマ` に進んでください。

## 7. トラブルシューティング

### 問題: AIが期待と違う出力をする

原因: 指示が曖昧、または情報不足

解決策:
```
具体的に伝える：
❌ 「もっと良くして」
✅ 「STEP2の手順をもっと詳しく、タップする場所まで書いて」
```

### 問題: 台本が長すぎる/短すぎる

原因: 時間配分の調整が必要

解決策:
```
「5分で収まるように短くして」
「もう少し詳しく、8分くらいの長さに」
```

### 問題: あらきりっぽくない

原因: 文体がずれている

解決策:
```
「もっとズボラ感を出して」
「『私みたいなズボラな人でも』というフレーズを入れて」
「語尾を柔らかくして」
```

### 問題: 専門用語が多い

原因: 40-60代向けの配慮が足りない

解決策:
```
「専門用語を噛み砕いて説明して」
「『アーカイブ』を『保管箱のようなもの』と言い換えて」
```

## 8. ファイル構成

### システム全体

```
YT_TalkScript/
├── CLAUDE.md          ← システム定義（自動読み込み）
├── MANUAL.md          ← このマニュアル（Markdown）
├── manual/
│   └── index.html    ← 同内容の読みやすいWeb版（自動更新しない運用）
│
├── config/           ← 設定ファイル（SSOT）
│   ├── brand.md      ← ブランド定義
│   ├── audience.md   ← 視聴者ペルソナ
│   ├── voice.md      ← 文体・トーン
│   └── quality.md    ← 品質基準
│
├── templates/        ← 台本テンプレート
│   ├── structure-A.md ← ツール活用型
│   ├── structure-B.md ← 機能紹介型
│   └── structure-C.md ← まとめ紹介型
│
├── agents/           ← エージェント定義
│   ├── scout.md       ← /テーマ調査（ネタ調査）
│   ├── planner.md    ← 企画
│   ├── architect.md  ← 構成
│   ├── writer.md     ← 執筆
│   ├── titler.md     ← タイトル・サムネ
│   └── reviewer.md   ← レビュー
│
├── examples/         ← 成功台本サンプル
│
└── outputs/          ← 成果物
    └── YYYYMMDD-テーマ/
        ├── 00-discovery.md  ← /テーマ調査を実行したとき（任意ステップのみ）
        ├── 01-plan.md
        ├── 02-structure.md
        ├── 03-script.md
        ├── 04-title.md
        └── 05-review.md
```

### 各ファイルの役割

| ファイル | 役割 | 編集頻度 |
|----------|------|----------|
| CLAUDE.md | システム全体の定義 | 低 |
| MANUAL.md / manual/index.html | 使い方の説明（明示指示があるときだけ更新） | 低〜（依頼時のみ） |
| config/*.md | ブランド・品質基準 | 中（改善時） |
| templates/*.md | 台本の型 | 低 |
| agents/*.md | 各工程の定義 | 低 |
| examples/*.md | 成功台本 | 高（追加） |
| outputs/ | 成果物 | 毎回 |
